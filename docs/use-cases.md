#### UC - 1
1. Название
	1. Редактирование сущностей администратором
2. Акторы
	1. Пользователь с ролью "Администратор"
3. Предусловие
	1. Администратор авторизован в системе
	2. Администратор находится на странице любой сущности
4. Триггер
	1. Нажатие кнопки "Редактировать"
5. Результат
	1. Сущность обновлена и доступна для просмотра с учетом внесенных изменений
6. Основной сценарий
	1. Пользователь переходит в режим редактирования сущности;
	2. Вносит изменения в нужный блок;
	3. Нажимает кнопку "Сохранить"
	4. Система успешно сохраняет изменения и не закрывает режим редактирования.
	5. Система успешно сохраняет в БД историю редактирования сущности.
7. Альтернативный сценарий
	1. Альтернативный сценарий A1 — выход без сохранения
		1. На шаге 2 основного сценария
		2. 3.a. Закрывает вкладку в браузере или переходит на другую сущность
		3. 4.a. Система спрашивает "Вы уверены, что хотите выйти? Не сохраненные изменения исчезнут" с вариантами ответа "Да" и "Остаться".
		4. 5.a. Пользователь нажимает "Да"
		5. 6.а. Система не сохраняет изменения.
	   
	4. Альтернативный сценарий A2 — отказ от выхода
		1. На шаге 4 альтернативного сценария А1 
		2. 4.b. Пользователь нажимает "Остаться".
		3. 5.b. Система оставляет пользователя на текущей странице редактирования.
		4. 6.b. Пользователь находится в режиме редактирования сущности.
	    
	9. Альтернативный сценарий А3 - ошибка сохранения.
		1. На шаге 3 основного сценария
		2. 4.c. Система не смогла сохранить изменения и отображает информативное сообщение об ошибке
		3. 5.c. Пользователь возвращается в режим редактирования
8. Постусловие
	1. Сущность сохранена
	2. Создана новая версия сущности.

#### UC - 2
1. Название
	1. Поиск сущностей автомобиля
2. Акторы
	1. Неавторизованный пользователь
	2. Авторизованный пользователь
3. Предусловие
	1. Пользователь находится в любом разделе веб-приложения
	2. В интерфейсе доступно поле поиска
4. Триггер
	1. Пользователь вводит поисковый запрос
5. Результат
	1. Пользователь получил релевантный результат поиска
6. Основной сценарий
	1. Нажимает на поле ввода
	2. Вводит три или более символов
	3. Система отображает ограниченное кол-во результатов
	4. Пользователь нажимает на нужный вариант
	5. Система открывает страницу сущности с выбранным вариантом
7. Альтернативный сценарий
	1. Альтернативный сценарий А1: "Введено менее 3-х символов"
		1. На шаге 2 основного сценария
		2. 3.а. Система не отображает результаты поиска

	2. Альтернативный сценарий А2: "Просмотр всех результатов"
		1. На шаге 3 основного сценария
		2. 4.б. Пользователь выбирает "Просмотреть все результаты"1.
		3. 5.б. Система открывает страницу всех результатов поиска.

	3. Альтернативный сценарий А3: "Нет результатов"
		1. На шаге 3 основного сценария
		2. 3.в. Система отображает сообщение об отсутствии результатов
8. Постусловие
	1. Пользователь либо перешёл к сущности, либо получил информацию об отсутствии результатов
